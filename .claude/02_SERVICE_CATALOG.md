# 02. 서비스 카탈로그 & 요금 정책

> rankfree가 제공할 서비스 목록·데이터 처리 방식·무료/유료 구분.
> 출처: 태훈님 기획 (2026-07-10). **초안 — 확정 전 검토 필요 항목은 ⚠️ 표시.**
> 수익 모델: 무료 분석으로 사용자 모집 → 유료 전환 + 마케팅 서비스 중개·광고.

## 처리 방식 3분류

| 방식 | 설명 | 대상 |
|---|---|---|
| **서버 수집** | 서버에서 PHP curl(네이버 pcmap GraphQL 등) / 공식 API 호출 | 플레이스 순위·분석, 키워드 트렌드, 블로그 |
| **확장 프로그램** | 크롬 확장 — 사용자 브라우저에서 수집(서버 크롤링 불가 영역) | **쇼핑 전반** |
| **설치형 프로그램** | 무료 배포 데스크톱 툴 (모객·리텐션용) | 블로그 이웃추가·댓글 작성기 |

## A. 플레이스 (네이버 스마트플레이스)

| # | 서비스 | 내용 | 데이터 소스 | 요금 | crm 이식 |
|---|---|---|---|---|---|
| A1 | **플레이스 순위체크** | 키워드별 내 플레이스 순위 조회·추적 | pcmap GraphQL (nCaptcha 토큰) | **무료 100개** + 추천인당 +20 (**최대 200개**), 초과 **유료** | 모듈1 ✓ |
| A2 | **순위체크 API** | 외부에 순위조회 REST 제공 | A1 엔진 | **유료** | 모듈5 ✓ |
| A3 | **내 플레이스 분석** | 내 업체 상세 신호(리뷰·저장·정보충실도 등) 1회성 진단 | pcmap + m.place SSR | **무료 1회성, 월 5회** | 모듈2/3 부분 ✓ |
| A4 | **플레이스 경쟁분석** | 키워드 상위 30 경쟁사 vs 내 매장 비교(리뷰·저장·예약·평점·점수) 1회성 | 모듈2 엔진 | **무료 1회성, 월 5회** | 모듈2 ✓ |
| A5 | **경쟁분석 추적** | A4를 매일 자동 기록·추이·알림 | 모듈2 + 스케줄러 | **유료** | 모듈2 크론 ✓ |
| A6 | **경쟁분석 추적 API** | A5 데이터 REST 제공 | A5 | **유료** | 신규(API 래핑) |

> 순위체크 카운트 = **추적 중인 (키워드×플레이스) 슬롯 수**로 해석. ⚠️ 정확한 카운트 단위(등록 슬롯 vs 일 조회수) 확정 필요.

## B. 키워드

| # | 서비스 | 내용 | 데이터 소스 | 요금 |
|---|---|---|---|---|
| B1 | **키워드 트렌드 분석** | 기간별·성별·연령별 검색량 및 비율 | **네이버 검색광고 API** + 크롤링 보강 | ⚠️ 무료/유료 미정 |

## C. 쇼핑 (⚠️ 서버 크롤링 불가 → **확장 프로그램**으로 수집)

| # | 서비스 | 내용 | 요금 |
|---|---|---|---|
| C1 | **쇼핑 시장 분석** | 키워드 분석 + **판매량·매출·수익 예측**. 수익예측 = 판매가·수수료·원가·인건비·부자재 입력 → 예상 수익 | ⚠️ 미정 |
| C2 | **쇼핑 리뷰 분석** | 인기 옵션, 자주 언급 단어 카운팅, 감정분석 | ⚠️ 미정 |
| C3 | **쇼핑 스토어 분석** | 특정 스토어 상품별 상세(판매량·예상매출·인기상품) | ⚠️ 미정 |
| C4 | **쇼핑 셀러력** | 키워드 노출 몰 vs 내 몰의 SEO·리뷰 등 경쟁 비교 | ⚠️ 미정 |

> 쇼핑은 전부 확장 프로그램이 브라우저에서 데이터를 긁어 서버로 전송 → 서버가 분석·저장. 확장 ↔ 서버 API 규약 별도 설계 필요.

## D. 블로그

| # | 서비스 | 내용 | 데이터 소스 | crm 이식 |
|---|---|---|---|---|
| D1 | **블로그 수집 — 품질** | 검색 관련도순 1~10페이지 노출 블로그의 품질 분석 | search.naver 블로그탭 + 블로그 지수 | 모듈4 부분 ✓ |
| D2 | **블로그 수집 — 아이디 추출** | 키워드 입력 → 월/주 단위로 검색일 바꿔가며 노출 블로그 **아이디 추출·중복제거** (블로그탭 기간검색 `nso=so:r,p:fromYYYYMMDDtoYYYYMMDD`) | search.naver 기간검색 | 신규 |
| D3 | **블로그 아이디 분석** | 특정 블로그 지수·활동·영향력 점수화 | 모듈4 `spb_fetch` | 모듈4 ✓ (crm 구현됨) |
| D4 | **무료 설치형 프로그램** | 블로그 **이웃추가기·댓글 작성기** (무료 배포 = 모객/리텐션) | 데스크톱 앱 | 신규 |

## 요금 구조 요약 (현재 확정분)

**무료 (모객)**
- 순위체크 100개 (추천인당 +20, 최대 200개)
- 내 플레이스 분석 월 5회 / 플레이스 경쟁분석 월 5회 (각 1회성)
- 블로그 아이디 분석·수집 (범위 ⚠️)
- 설치형 프로그램(이웃추가·댓글)

**유료 (전환)**
- 순위체크 200개 초과 / 순위체크 API
- 경쟁분석 추적 + 추적 API
- 쇼핑 분석군(확장 프로그램) ⚠️ 요금 미정
- 키워드 트렌드 ⚠️

**중개·광고 (수익 3축)**
- 플레이스 최적화·블로그 체험단·광고 대행 중개

## ⚠️ 확정 필요 항목
1. 순위체크 "100개" 카운트 단위 (추적 슬롯 수 vs 일 조회 횟수)
2. 추천인 리워드 지급/차감 규칙 (탈퇴·어뷰징 방지)
3. 키워드 트렌드·쇼핑군 유·무료 및 가격
4. 유료 요금제 형태 (구독 월정액 vs 크레딧 종량)
5. 확장 프로그램 배포 방식(크롬 웹스토어 심사) 및 서버 연동 인증
6. 설치형 프로그램의 법적/약관 리스크(네이버 자동화 정책) 검토
